# Face Enrollment — Demo Patients

Эта папка содержит эталонные фотографии лиц для face-ID на демо-показе.
AI сравнивает live-видео с этими фото через cosine similarity (Mediapipe FaceMesh embeddings).

## Что положить сюда

Для каждого демо-пациента — **один файл** с именем формата:

```
<patient-id>.jpg
```

## Требования к фото

✅ **Frontal**: лицо смотрит прямо в камеру
✅ **Хорошее освещение**: дневной свет или рассеянная лампа
✅ **Нейтральное выражение**: можно лёгкая улыбка, без широкой
✅ **Без очков и масок** (если на демо будут — то и на эталоне)
✅ **Минимум 720×720 px** (квадрат предпочтительнее)
✅ **JPEG, < 500 KB** (сжать через TinyJPG если нужно)
❌ **Без фильтров и ретуши** (Instagram-фильтры путают AI)
❌ **Не selfie-зеркалирование** — нормальная фронтальная съёмка

## Как процессить

После добавления фото запусти:

```bash
pnpm enroll:faces
```

(скрипт сгенерирует embeddings → `public/face-enrollment/embeddings.json`).

На демо AI берёт live-кадр → считает embedding → сравнивает со всеми
embeddings.json → выбирает максимум → если cosine similarity > 0.65
→ match подтверждён.

## Privacy / GDPR / ZRU-547

Эти фото — для DEMO. На production должны храниться в Supabase Storage
(не в git!) с RLS, шифрованием at-rest, и сроком хранения 30 дней.
В git коммитим только `.gitkeep` — фото добавь локально, они в `.gitignore`.

## Если фото нет

AI работает в "trust mode": face check проходит автоматически с confidence 0.5
и **флаг `face_unenrolled`** для врача. Демо не блокируется.
